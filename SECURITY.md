# Security Policy

## Supported Versions

| Version | Supported |
|---------|-----------|
| Latest `main` branch | ✅ Yes |
| Older commits | ❌ No |

## Reporting a Vulnerability

If you discover a security vulnerability in LYF Mail Docs or any related repository:

1. **Do NOT open a public issue.**
2. Email **contact@lyfmail.com** with details.
3. Include steps to reproduce, impact assessment, and suggested fixes if available.
4. Allow up to 72 hours for initial response.

## Disclosure Policy

- We will acknowledge receipt within 72 hours.
- We will investigate and provide updates within 7 days.
- Once resolved, we will publish a security advisory and credit the reporter (with permission).

## Security Best Practices for Documentation

- All code examples should use HTTPS.
- No hardcoded credentials or API keys in examples.
- Dependencies should be kept up to date.
- Sensitive deployment configurations should be clearly marked.
