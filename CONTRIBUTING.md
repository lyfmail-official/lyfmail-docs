# Contributing to LYF Mail Docs

Thank you for your interest in contributing to the LYF Mail documentation! This guide will help you get started.

## How to Contribute

### Reporting Issues
- Use GitHub Issues to report documentation errors, broken links, or outdated information.
- Provide clear descriptions, screenshots where applicable, and suggested fixes.

### Suggesting Improvements
- Open a GitHub Issue with the `enhancement` label.
- Describe the improvement, why it's needed, and how it benefits users.

### Submitting Documentation
1. **Fork** the repository.
2. **Create a branch** (`git checkout -b docs/your-topic`).
3. **Write your documentation** in Markdown format.
4. **Test all code examples** and verify all links.
5. **Submit a Pull Request** with a clear description.

## Writing Guidelines

- Use **clear, simple language** — avoid jargon where possible.
- Include **code examples** for all technical instructions.
- Use **proper Markdown formatting** — headers, lists, code blocks, tables.
- Add **screenshots or diagrams** where visual explanation helps.
- Follow the **existing directory structure** and file naming conventions.
- Ensure **all external links are valid** and point to HTTPS resources.

## Code of Conduct

All contributors must follow our [`CODE_OF_CONDUCT.md`](CODE_OF_CONDUCT.md). Be respectful, constructive, and inclusive.
